- Tool: **Frama-C 25.0-beta (Manganese)**, EVA value analysis (`frama-c -eva -eva-slevel 0`).
- Host: Ubuntu 24.04.3, kernel 6.17.0-19-generic, aarch64 — the same VM as the eBPF witness.
- Full log: `out/eva_slevel0.log` (verbatim). Command: `bash run.sh`.

## What it establishes

1. **A-opacity on a real analyzer (Definition 2).** For the working mod-3 NAND gate, EVA's
   inferred value for the output is `out ∈ {0; 1}` — the full Boolean range, i.e. `⊤`. A sound,
   widely-used value analysis that we did not write certifies *nothing* about the output bit,
   though it depends on the inputs. Because EVA is join-based here, this `⊤` is a single top
   value — no path-sensitivity caveat, unlike the (path-sensitive) eBPF verifier.

2. **Non-triviality / localization (localization / repair).** The **same** analysis certifies the
   ablation as `abl ∈ {1}` (modulus 7 degenerates the gate to the constant 1). So EVA is not
   trivially blind; the blindness is localized to the working `mod`-3 channel — an operation for
   which the interval/Cvalue domain is below its complete shell.

3. **Not an artifact of imprecision elsewhere.** `acc ∈ {1; 2; 3}` in both gates confirms the
   inputs are correctly constrained to `{0,1}`, and the run raises **0 alarms**. The opacity is
   the designed incompleteness of the sound domain at `mod`, nothing else.

## System-independence, closed

Two structurally different sound analyzers now exhibit the same phenomenon on the same abstract
structure (paper §9 outlook):

| | analyzer | style | channel φ | output abstraction |
|---|---|---|---|---|
| eBPF witness | Linux in-kernel verifier | path-sensitive | hash-map occupancy | `⊤` (join over paths; verifier forks) |
| this witness | Frama-C EVA (Cvalue) | join-based | `acc mod 3` | `out ∈ {0;1}` = `⊤` (one top value) |

The phenomenon therefore tracks *sound-but-incomplete abstraction*, not an eBPF idiosyncrasy —
which is exactly what the abstraction-gap thesis predicts.
